Плохо ИИ справлялся с этим 
Переполнение по высоте: на маленьких экранах (iPhone SE) блоки “влезают” на Pro Max, но обрезаются/наезжают друг на друга на SE.
Safe Area/челка/нижний индикатор: ИИ часто ставит фиксированные paddingTop/paddingBottom, из-за чего контент заезжает под системные зоны.
Неправильный баланс “без скролла / со скроллом”: либо добавляет скролл везде, либо убирает его там, где он критичен (например, список).
Слишком “вебовые” решения: фиксированные ширины, некорректная адаптация карточек и кнопок под узкий viewport.
Переоценка UI-библиотек: отдельные компоненты на web могут вести себя иначе (warning’и, кликабельность, deprecated props).
Как я это контролировал, чтобы не сломать приложение:

Ввел Safe Area как обязательный слой: SafeAreaProvider + useSafeAreaInsets() на ключевых экранах.
Разделил поведение по типу экрана:
Paywall — без скролла, но с compact-режимом по высоте (useWindowDimensions), чтобы ужимать отступы/типографику.
Meditations — скролл только у списка (FlatList), чтобы не ломать UX.
Убрал нестабильные внешние зависимости (сетевые картинки), потому что из-за таймаутов визуально “плыла” верстка.
Явно зафиксировал логику подписки через isSubscribed и проверку requiresPremium && !isSubscribed — чтобы UI-блокировки были предсказуемыми.
После каждой существенной правки прогонял проверки и ручные сценарии (переходы, клики, выбор настроения, lock-state).
